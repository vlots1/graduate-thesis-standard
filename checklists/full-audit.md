# Full Audit Checklist

Use this as an inspection sequence, not as a generic report. Read its source rule file before recording a finding.

## Components and assembly

- [ ] Required cover, title page, declarations, abstracts/keywords, contents, main text, references, achievements, acknowledgements, and committee list exist in prescribed order.
- [ ] Conditional lists/symbol table are present when the document evidence shows they are needed.
- [ ] Optional appendix/index, if present, are located and labelled correctly.

## Layout and hierarchy

- [ ] A3 cover and A4 inner pages; inner-page margins match the rule.
- [ ] Printing/pages, section breaks, headers, Roman front matter, Arabic main matter, and page-number placement are verified from rendered output or marked unavailable.
- [ ] Chapters start new pages; headings follow one permitted hierarchy and contents matches it.
- [ ] Main text, captions, equations, footnotes, references, and appendices use their applicable layout rules.

## Objects and references

- [ ] Figure/table/equation sequences, placements, captions, list entries, and in-text mentions agree.
- [ ] Appendix object numbering is separate.
- [ ] Citation style is one GB/T 7714 system; in-text citations and reference records are bijective.

## Semantic consistency

- [ ] Abstract pairs align in purpose, method, result, conclusion, sample size, and keywords.
- [ ] Terms, abbreviations, variables, groups, numbers, and claims agree across sections.
- [ ] First-use explanations and legal units are checked or listed for manual review.

## Format-specific checks

- [ ] DOCX: styles, field-based contents/captions, direct formatting, headers/footers, section breaks, and page fields.
- [ ] LaTeX: `bnuthesis` entry, output PDF, bibliography setup, labels/references, local overrides, and template-core changes.
